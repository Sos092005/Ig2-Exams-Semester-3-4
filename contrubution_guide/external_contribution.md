# Création d’un Patch

Les patchs (problèmes) peuvent être utilisés pour suivre les améliorations, les erreurs ou d’autres demandes.

## Création d’un patch (problème) à partir d’un dépôt

1. Sur GitHub, accédez à la page principale du dépôt.

2. Sous le nom de votre référentiel, cliquez sur **<svg version="1.1" width="16" height="16" viewBox="0 0 16 16" class="octicon octicon-issue-opened" aria-label="issue-opened" role="img"><path d="M8 9.5a1.5 1.5 0 1 0 0-3 1.5 1.5 0 0 0 0 3Z"></path><path d="M8 0a8 8 0 1 1 0 16A8 8 0 0 1 8 0ZM1.5 8a6.5 6.5 0 1 0 13 0 6.5 6.5 0 0 0-13 0Z"></path></svg> Problèmes**.

   ![Capture d’écran de la page principale d’un dépôt.](/contribution_guide/assets/images/repo-issues.png)

3. Cliquez sur **Nouveau problème**.

4. Si votre dépôt utilise des modèles de problème, à côté du type de problème que vous souhaitez ouvrir, cliquez sur **Démarrer**.

   Si le type de problème que vous souhaitez ouvrir n’est pas inclus dans les options disponibles, cliquez sur **Ouvrir un problème vide**.

![Capture d’écran du sélecteur de modèle pour un problème. Sous les choix de modèle se trouve un lien intitulé « Ouvrir un problème vide », encadré en orange foncé.](/contribution_guide/assets/images/blank-issue-link.png)

1. Dans le champ « Titre », tapez un titre pour votre problème.

2. Dans le champ du corps du commentaire, tapez une description de votre problème.
   Pour faire référence à une discussion connexe, collez l’URL de la discussion dans la description du problème.

> \[!TIP]
> Au fur et à mesure de votre saisie, GitHub peut suggérer d’éventuels problèmes en double déjà présents dans le référentiel. Si une suggestion semble pertinente, vous pouvez cliquer sur le problème existant au lieu de en créer une nouvelle. Ces suggestions s’affichent une fois le titre rempli et le corps atteint 100 caractères, et jusqu’à trois problèmes existants peuvent être affichés. Les suggestions ne sont pas bloquantes et ne vous empêchent pas de créer votre problème.

1. Lorsque vous avez terminé, cliquez sur **Envoyer le nouveau problème**.
